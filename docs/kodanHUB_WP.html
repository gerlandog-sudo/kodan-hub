Arquitectura Técnica General

Descripción del stack tecnológico, arquitectura en 4 capas, modelo de seguridad y sistema de tokens KDN.

1.1 Arquitectura en 4 Capas

KodanHUB implementa un patrón Proxy-Pass donde las aplicaciones cliente nunca poseen directamente API Keys de proveedores de IA. La comunicación fluye a través de 4 capas: Clientes, Gateway, Proveedores y Administración.

1.4 Sistema de Tokens KDN

KodanHUB implementa un sistema de tokens propio llamado KDN Token, diseñado para simplicidad y operación sin estado compartido. No utiliza JWT.

Formato del Token

KDN-[PREFIX]-[16-CHAR HEX HASH]
Ejemplo:
KDN-SC-8DAC5109A1508665
KDN-TT-B3F2A1C4D5E6F789

Algoritmo de Generación

function generateKodanToken($name) {
    $prefix = initials($name);
    $hash = strtoupper(
        substr(md5(uniqid()), 0, 16)
    );
    return "KDN-$prefix-$hash";
}
